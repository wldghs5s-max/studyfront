이벤트

1. 이벤트 모델의 종류

1) 고전 이벤트 모델

요소 객체가 가지고 있는 이벤트 속성에 이벤트 핸들러를 연결하는 방법이다.
이벤트 객체를 제거할 수 있는데 제거하고자 한다면 해당 이벤트 속성 값에 null을 대입하면 된다.

2) 인라인 이벤트 모델

요소 내부에 이벤트 속성으로 실행 내용을 작성하는 방법이다.
단순한 코드의 경우엔 직접 코드를 입력을 해도 좋으나 script 태그 내에 정의된 함수를 호출하는 방식을 선호한다.

3) 표준 이벤트 모델

w3에서 공식적으로 지정한 이벤트 모델로 한 번에 여러 개의 이벤트 핸들러 설정이 가능하다.

2. 이벤트가 발생한 요소 객체에 접근하는 방법

이벤트가 발생한 요소 객체는 내부적으로 이벤트 발생 정보를 담고 있는 객체를 매개변수로 전달한다.
이벤트 객체를 통해서 해당 요소 객체에 접근이 가능하고 콜백 함수의 종류에 따라 this로 접근이 가능하다.

3. 태그별 기본적으로 가지고 있는 이벤트 제거

기본 이벤트를 가지고 있는 요소 a : 클릭 시 href 속성에 제시되어 있는 페이지를 요청하는 기본 이벤트 보유 submit : 클릭 시 입력 양식 제출 후 페이지를 다시 로딩하는 기본 이벤트 보유 기본 이벤트가 있는 요소에 onclick 이벤트를 설정하면 이 코드가 우선적으로 먼저 실행되고 그 다음에 기본 이벤트가 실행된다. 태그에 기본적으로 설정되어 있는 이벤트를 제거하기 위해서는 이벤트 핸들러의 return 값을 false로 하면 된다.

기본 이벤트 제거는 return false나 evt.preventDefault()를 이용하여 제거 가능하다
